# 第8章 発表メモ

## 重要なポイント

### 歴史的背景
- 1964年: Peter J. Landinがクロージャを発明
- 1975年: SussmanとSteeleがSchemeでクロージャの定義を確立
- Ruby: 20年以上前の研究をエレガントに実装

### クロージャの定義
> ラムダ式と、そのラムダ式が引数に適用されるときに使われる環境の両方を持つデータ構造

### rb_block_t 構造体
```c
typedef struct rb_block_struct {
    VALUE self;           // ブロック参照時のselfポインタ
    VALUE klass;          // 現在のオブジェクトクラス
    VALUE *ep;            // 環境ポインタ（重要！）
    rb_iseq_t *iseq;     // YARV命令列
    VALUE proc;           // Procオブジェクト参照
} rb_block_t;
```

### 強調すべき点
1. ブロックの「二重人格」
   - メソッドのような独立性
   - 親スコープへのアクセス能力

2. メモリ管理の巧妙さ
   - スタックからヒープへのコピー
   - EPのリセット機能

3. パフォーマンスとの天秤
   - 71%のオーバーヘッド
   - 実用的には影響限定的

## 実験結果の詳細

### while vs each ベンチマーク
- while ループ: 0.44秒（100万回実行）
- each ブロック: 0.76秒（100万回実行）
- 差異: 約71%の実行時間増加

### 理由
- ブロック呼び出しのオーバーヘッド
- rb_block_t構造体の作成
- 新しいスタックフレームの作成
- EPの設定

## 質疑応答想定

### Q1: なぜブロックを使うのか？
A: 可読性とエレガンス。実用的なアプリケーションではパフォーマンス影響は限定的

### Q2: lambdaとprocの使い分けは？
A: 引数チェックの厳密さが違う。is_lambdaフラグで制御

### Q3: 他の言語との比較は？
A: JavaScriptのクロージャ、Pythonのラムダなど。Rubyの実装は特にエレガント
