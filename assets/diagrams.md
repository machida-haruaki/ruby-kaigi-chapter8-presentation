### 2. スタックフレーム変化図
```
最初の状態:
┌─────────────────┐
│ トップレベル     │
│ スタックフレーム │
│ locals: str     │
└─────────────────┘

10.times呼び出し後:
┌─────────────────┐
│ ブロック用       │ ←新しく作成
│ スタックフレーム │
│ locals: str2    │
├─────────────────┤
│ times メソッド   │
│ スタックフレーム │
├─────────────────┤  
│ トップレベル     │
│ スタックフレーム │ 
│ locals: str     │
└─────────────────┘
```

### 3. クロージャの概念図
```
クロージャ = ラムダ式 + 環境

┌─────────────┐   ┌─────────────┐
│  コード片   │ + │    環境     │
│  (iseq)     │   │   (EP)     │
│             │   │            │
│ puts str    │   │ str="..."  │
└─────────────┘   └─────────────┘
        ↓
   rb_block_t構造体
```

### 4. rb_block_t構造体図解
```
rb_block_t
├── iseq: rb_iseq_t*  (YARV命令列) ←「ラムダ式」
├── ep: VALUE*        (環境ポインタ) ←「環境」
├── self: VALUE       (selfポインタ)
├── klass: VALUE      (クラス情報)
└── proc: VALUE       (Procオブジェクト)
```

### 5. メモリ配置図（スタック vs ヒープ）
```
スタック                    ヒープ
┌─────────────┐              ┌─────────────────┐
│ locals: str │ ──────────→ │ RString          │
│             │              │ "quick brown..." │
│ EP ─────────┼──┐          └─────────────────┘
└─────────────┘  │           
                 │           ┌─────────────────┐
                 └─────────→ │ rb_env_t        │
                             │ (ヒープコピー)   │
                             └─────────────────┘
```


### 8. 設定と実行の分離概念図
```
従来のパラダイム:
┌──────────────────────────┐
│ 関数呼び出し               │
│ = 設定 + データ (毎回)     │
└──────────────────────────┘

クロージャのパラダイム:
┌─────────────┐   ┌─────────────┐
│ 関数作成     │   │ 関数呼び出し   │
│ = 設定込み   │ → │ = データのみ   │
│ (一度だけ)   │   │ (何度でも)     │
└─────────────┘   └─────────────┘
```

## 作成時の注意点

1. **シンプルで視覚的**: 複雑すぎない、一目で理解できる
2. **統一感**: フォント、色使い、レイアウトを統一
3. **重要部分の強調**: 太字、色、矢印で重要な部分を明確に
4. **投影時の見やすさ**: 大きな文字、コントラストを重視
5. **第8章PDFとの連携**: 元の図表も効果的に活用
